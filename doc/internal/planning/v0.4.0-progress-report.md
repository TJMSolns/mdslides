## User-Facing Features

### Directory-Based Themes

**Before (v0.3.0):**
```
themes/retisio.json  ← Single file, no assets
```

**After (v0.4.0):**
```
themes/retisio/
├── theme.json         ← Configuration
└── backgrounds/       ← Assets included
    └── *.png
```

**Benefits:**
- Self-contained (zip and share)
- Clear organization
- Portable across systems

### Template-Specific Backgrounds

**Theme Configuration:**
```json
{
  "templateBackgrounds": {
    "title": "backgrounds/title-page.png",
    "content": "backgrounds/content-page.png"
  }
}
```

**Result:**
- Title slides automatically get title background
- Content slides automatically get content background
- No per-slide configuration needed

### Theme Loading

**Command:**
```bash
mdslides render slides.md output.html --theme retisio
```

**Output:**
```
Loading theme: retisio
✓ Loaded theme: the prior organization v1.0.0
```

**Error Handling:**
```

Searched for: ./themes/nonexistent/theme.json

Available themes:
  - retisio

Use 'mdslides config set theme-dir /path' to change theme directory.
```

## Lessons Learned

### What Went Well ✅
1. **TDD Discipline:** Writing tests first caught several issues early
2. **Ceremony Process:** PDRs and Event Storming prevented scope creep
3. **Incremental Progress:** Small, testable chunks made progress visible
4. **Error Messages:** Clear errors saved debugging time
5. **End-to-End Testing:** Tutorial presentation validated entire feature

### Challenges Overcome 🔧
1. **Cats Effect Syntax:** Required learning `adaptError` and `fromEither`
2. **Circe Decoder:** Custom Theme decoder needed for optional templateBackgrounds
3. **Template Availability:** Tutorial had to use only existing templates (title, content)
4. **Path Resolution:** Careful handling of theme-relative vs absolute paths

### Process Improvements 💡
1. **Parallel Testing:** Could have run domain + infrastructure tests in parallel
2. **Template Creation:** Should add section-title, diagram, closing templates soon
3. **Documentation:** Keep tutorial in sync with available features
